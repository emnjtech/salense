# AI Narrative Generation

Salense AI follows the AIES sequence:

Business Reasoning -> Context Builder -> Prompt Orchestrator -> AI Abstraction Layer -> Provider Adapter -> LLM Narrative Generator -> Today Dashboard

The LLM is a communication layer only. It does not calculate analytics, discover business facts, query marketplace APIs, or make autonomous commercial decisions.

## Environment

Set these variables in the API environment to enable OpenAI narrative generation:

```powershell
$env:OPENAI_API_KEY="sk-your-key"
$env:OPENAI_MODEL="gpt-4o-mini"
$env:TEMPERATURE="0.2"
```

If `OPENAI_API_KEY` is missing, the API returns the deterministic executive summary generated by the Business Reasoning System.

## Context Objects

The Context Builder only passes sanitized reasoning output to the provider:

- business overview
- business health
- observations
- diagnostics
- recommendations
- forecasts
- confidence
- explainability references

The context must not contain raw marketplace payloads, credentials, tokens, hashes, encrypted values, or database records.

## Prompt Versioning

Sprint 3 uses `prompt-v1` templates for:

- Daily Briefing
- Executive Summary
- Recommendation Explanation
- Business Query

Prompts instruct the LLM to explain, summarize, organize, and communicate verified deterministic findings only.

## Provider Architecture

Application code calls `AIService`, which resolves a provider through `ProviderFactoryService`.

Only provider adapters may communicate with external LLM APIs. `OpenAIProvider` is the first adapter. Future adapters such as Anthropic, Gemini, or Azure OpenAI should implement the same `IAIProvider` contract without changing the reasoning engines.

## Fallback

If OpenAI is unavailable, unconfigured, times out, or returns an invalid response, the narrative generator falls back to deterministic wording. The Today dashboard should continue to function.
